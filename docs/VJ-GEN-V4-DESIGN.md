## 三、AI 工作流节点

### 节点类型

| 节点类型 | 图标 | 功能 | 输入 | 输出 |
|---------|------|------|------|------|
| 音频输入 | 🎵 | 加载音频文件 | - | 音频URL |
| BPM检测 | 📊 | 分析节拍/能量 | 音频URL | 节拍数据 |
| 歌词分析 | 📝 | NLP歌词→时间轴 | LRC文件 | 歌词片段列表 |
| Prompt生成 | ✨ | LLM生成视觉Prompt | 歌词+情绪 | 提示词 |
| 风格选择 | 🎨 | 选择视觉风格预设 | - | 风格参数 |
| 静帧生成 | 🖼️ | AI生成关键帧 | Prompt | 图像URL |
| 视频生成 | 🎬 | 图生视频 | 图像+Prompt | 视频URL |
| 时间编排 | ⏱️ | 片段拼接+转场 | 片段列表 | 时间线 |
| 渲染输出 | 📺 | 最终合成导出 | 时间线 | 视频文件 |

### 节点连接规则

```
[🎵音频] → [📊分析] → [📝歌词] → [✏️Prompt] → [🖼️生成] → [⏱️编排] → [🎬视频] → [📺输出]
                                      ↑
                               [🎨风格]─┘
```

## 四、核心操作流程

### 4.1 自动模式（一键生成）
1. 导入音频文件
2. 系统自动完成：BPM检测 → 歌词分析 → Prompt生成 → 静帧生成 → 时间编排 → 视频生成
3. 预览 → 调整 → 导出

### 4.2 手动模式（精细控制）
1. 音频 → 分段标记
2. 每段手动写Prompt或调整AI生成
3. 选择风格预设
4. 逐个生成静帧
5. 调整时间线
6. 生成视频片段
7. 最终合成

## 五、技术实现

### 5.1 前端技术栈
- **框架**: Next.js 14 + React 18
- **样式**: Tailwind CSS + CSS Variables
- **画布**: 自研 Canvas 或 Fabric.js
- **状态管理**: Zustand / Redux Toolkit
- **拖拽**: @dnd-kit / react-dnd

### 5.2 节点流程实现
- **节点库**: 自研或复用 xyflow/react-flow
- **画布**: 支持缩放、平移、网格吸附

### 5.3 通信协议
- **WebSocket**: 实时预览、进度推送
- **REST API**: 常规CRUD操作
